# KuluoBishi (推箱子)

基于 Unreal Engine 5.5 的推箱子益智游戏，内置关卡编辑器，支持自定义关卡的创建与游玩。

## 试玩说明

### 基本操作

- **WASD / 方向键** — 控制角色在网格上移动（上下左右四方向）
- 角色面前有箱子时，会自动推动箱子前进
- **撤销（Undo）** — 走错了可以撤回上一步
- **暂停菜单** — 可重新开始或返回关卡选择

### 游戏目标

将所有箱子推到目标点（Goal）上即可通关。

### 关卡机制

| 地块类型 | 说明 |
|---------|------|
| **地板（Floor）** | 普通可通行地块 |
| **墙壁（Wall）** | 不可通行的障碍 |
| **目标点（Goal）** | 箱子的目的地，所有目标被占满即通关 |
| **压力板（Pressure Plate）** | 箱子放上后触发机关，可联动开启门 |
| **门（Door）** | 默认关闭阻挡通行，由压力板组控制开关 |
| **冰面（Ice）** | 角色和箱子会在冰面上持续滑行，直到碰到障碍物停下 |

### 关卡模式

- **预设关卡** — 游戏自带的关卡，按顺序解锁
- **自定义关卡** — 玩家通过编辑器创建的关卡

## 关卡编辑器说明

从主菜单进入关卡编辑器，可以自由设计推箱子关卡。

### 工具栏

| 操作 | 说明 |
|------|------|
| **新建（New）** | 创建指定尺寸的空白关卡 |
| **保存（Save）** | 将当前关卡保存为 JSON 文件 |
| **加载（Load）** | 加载已有的自定义关卡 |
| **测试（Test）** | 直接进入游玩模式测试当前关卡 |
| **返回（Back）** | 返回主菜单 |

### 笔刷系统

侧边栏提供 9 种笔刷，左键绘制，右键擦除：

- **Floor** — 铺设地板
- **Wall** — 放置墙壁
- **Ice** — 铺设冰面
- **Goal** — 放置目标点
- **Door** — 放置门（需要关联压力板组）
- **Pressure Plate** — 放置压力板
- **Box Spawn** — 设置箱子初始位置
- **Player Start** — 设置玩家出生点（唯一）
- **Eraser** — 擦除地块

### 分组机制

门和压力板通过"组"进行关联：

1. 放置门后，进入 **PlacingPlatesForDoor** 模式，为该门关联压力板
2. 同一组内的所有压力板被触发后，对应的门将打开
3. 通过 **Group Manager** 面板管理所有分组，支持创建、删除和自定义颜色

### 关卡验证

保存或测试时会自动验证关卡的合法性，检查包括：
- 是否存在玩家出生点
- 箱子与目标点数量是否匹配
- 门是否正确关联了压力板组
- 所有元素是否可达

## 设计思路

### 网格离散化

所有游戏逻辑在离散网格上运行，位置统一使用 `FIntPoint` 表示。GridManager 作为中枢管理网格状态、占位关系和移动验证，将复杂的空间逻辑简化为整数坐标运算。

### 组件化机制系统

游戏机制（门、压力板、目标点等）采用组件模式实现，均继承自 `GridMechanismComponent`，挂载在 `TileVisualActor` 上。新增机制只需编写新组件，无需修改已有代码，符合开闭原则。

### 事件驱动解耦

子系统之间通过 Unreal 多播委托通信（如 `OnActorLogicalMoved`、`OnPitFilled`）。移动、动画、计步、胜负判定等逻辑彼此独立，互不依赖。

### 撤销/快照栈

每次移动后在 GameState 中压入完整的关卡快照（`FLevelSnapshot`），撤销时弹出恢复。实现简单可靠，天然支持任意步数回退。

### 编辑器与游玩分离

编辑器和游玩使用各自独立的 GameMode（`LevelEditorGameMode` / `SokobanGameMode`），共享同一套 Grid 系统和数据格式。编辑器的"测试"功能直接序列化当前状态并切换到游玩模式，实现无缝衔接。

### 数据驱动

关卡以 JSON 格式存储，通过 `LevelSerializer` 读写。预设关卡和自定义关卡使用相同的数据格式和加载流程，编辑器产出的关卡可以直接被游戏读取。
