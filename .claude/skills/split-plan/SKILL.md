# 拆分计划为并行模块

## 输入
用户提供一个计划文件路径: `$ARGUMENTS`

如果用户没有提供路径，询问用户要拆分哪个计划文件。

## 执行步骤

### Step 1: 分析原始计划

阅读计划文件，提取以下信息：
- **所有组件/类/模块** 的清单
- 每个组件的**职责**和**公共接口**（方法、delegate、属性）
- 组件之间的**依赖关系**（编译依赖 vs 运行时依赖）
- 共享的**类型/枚举/结构体**

同时阅读计划中引用的已有代码文件，理解现有的 API 和数据结构。

### Step 2: 构建依赖图

分析组件间的依赖类型：
- **无依赖**: 组件只使用共享类型，不引用其他组件的头文件 → 可并行
- **编译依赖**: 组件 A 的头文件中 #include 了组件 B → A 必须在 B 之后
- **运行时依赖**: 组件 A 在运行时调用组件 B 的方法，但头文件不互相引用 → 可并行开发，集成时串联

**关键原则: 通过 delegate/接口解耦，最大化并行度。**

- 子组件之间不直接持有对方指针，而是暴露 delegate + Refresh 方法
- 由一个集成组件（如 MainWidget）统一绑定 delegate，编排交互流程
- 这样所有子组件可以并行开发，集成组件最后开发

### Step 3: 分组为并行模块

按以下规则分组：
1. **紧密耦合**的组件放入同一模块（如 ListView + ListItem）
2. **无依赖**的模块归入同一并行层
3. **集成组件**单独成为一个模块，放在最后一层
4. **已有代码的修改**可拆分为：无依赖部分（层1）+ 依赖集成的部分（最后层）

目标: 层1 的模块数量最大化（并行度最高）

### Step 4: 为每个模块生成详细计划

每个模块计划文件必须包含以下 sections：

```markdown
# Module {X}: {模块名}

## 概述
{一句话描述模块职责}

## 前置依赖
- {列出编译依赖和运行时依赖，标注哪些可以并行}

## 产出文件
{列出所有要创建/修改的文件路径}

## 类定义
{完整的类声明，包含:}
- 所有 UPROPERTY / UFUNCTION 声明
- 所有 delegate 声明（供外部绑定）
- 所有 public 方法（供外部调用）
- 所有 BindWidget 成员（UI 绑定点）
- 所有 protected/private 成员和方法

## 布局规格（如果是 UI 组件）
{ASCII 图示布局，标注尺寸、对齐方式、Visibility 规则}

## 交互逻辑
{用伪代码/流程图描述每个方法的行为，包括:}
- 用户操作 → delegate 广播 → 外部处理 → 回调更新 的完整链路
- 边界情况和错误处理
- 状态切换逻辑

## 验收标准
{可测试的验收条件列表}
```

**重要规则:**
- 类定义必须足够完整，让 subagent 可以直接照着写代码
- 交互逻辑必须覆盖所有用户操作路径，不留模糊空间
- 每个模块只引用共享类型头文件 + 自己模块内的头文件，不引用其他模块

### Step 5: 生成总计划（Master Plan）

创建一个总计划文件，包含：

1. **依赖图** — ASCII 图示并行层结构
2. **模块清单** — 表格列出每个模块的名称、Widget 数、并行层、详细计划链接
3. **执行计划** — 按并行层描述：
   - 每层启动哪些 subagent
   - 每个 subagent 的任务描述
   - 层与层之间的等待条件
4. **共享类型引用** — 列出所有模块共享的头文件和类型
5. **文件目录结构** — 最终产出的文件树
6. **Subagent 启动模板** — 给主 agent 的 prompt 模板
7. **注意事项** — 实现中的关键技术点和坑

### Step 6: 输出

将文件写入与原始计划相同的目录：
- 每个模块: `{PlanDir}/{PlanPrefix}_Module_{Letter}_{Name}.md`
- 总计划: `{PlanDir}/{PlanPrefix}_Plan.md`

其中 `{PlanPrefix}` 从原始计划文件名推导（去掉 .md 后缀）。

## 质量标准

- 每个模块计划应该是**自包含**的：subagent 只需阅读该文件 + 共享类型头文件就能完成开发
- 总计划应该是**可执行**的：主 agent 照着启动 subagent 就能完成整个开发
- 模块间的接口必须**精确匹配**：模块 A 暴露的 delegate 签名 = 模块 B 绑定的 delegate 签名
- 并行度尽量**最大化**：优先解耦而非合并
